A dangerous place to get some flow-based devgramming done.

Mission Statement

To do the things that one wants to do. To explore areas of language, code, and blueprints. To work hard on things that need to be worked on. To challenge oneself to achieve greater. To fall asleep knowing that an impact was made. To become a fully autonomous full-stack developer and beyond.

Project Goals

1) a lexer, parser, and interpreter that transpiles (inomlang + autoide) into a new project based on a node.js template that can be deployed to production.

2) a devgrammer portfolio in 100% plain old html, css, and javascript.

3) a backend server that manages ERP database records in PostgreSQL, user notifications and member 2 member personal messaging in RabbitMQ, and integrates with 3rd parties like stripe for payments and twilio for sms.

4) a frontend client that has an admin area for managers and a products area for customers. it looks like a mobile app but it runs on the web.

Target Audience

Someone that is interested in flow-based programming with a developer mindset. They think in terms of nodes, anchors, and edges. They have an itching desire to make impovements. They are interested in seeing their code visually through diagram technology when it comes to building a program; yet, terminal user interfaces are their friend.

How to use

Verify node.js package manager: 
$ npm --version
Then install this programming language: 
$ npm install --global inom
Read the manual for documentation: 
$ inom --help
Deploy this runtime backend to the internet: 
https://github.com:subvind/isisis
Download this desktop IDE frontend: 
https://github.com/subvind/autoide
