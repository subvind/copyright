A dangerous place to get some flow-based devgramming done.

Mission Statement

To do the things that one wants to do. To explore areas of language, code, and blueprints. To work hard on things that need to be worked on. To challenge oneself to achieve greater. To fall asleep knowing that an impact was made. To become a fully autonomous full-stack developer and beyond.

Project Goals

1) a text parser that converts code or inomscript to runnable rustlang for flow-based devgramming.

2) a devgrammer portfolio in 100% plain old html, css, and javascript.

3) a backend server that manages database records, quorum queues via asynchronous messaging, and inomscript package manager.

4) a frontend client that is diagram based, that is terminal based, and runs locally.

Target Audience

Someone that is interested in flow-based programming with a developer mindset. They think in terms of nodes, anchors, and edges. They have an itching desire to make impovements. They are interested in seeing their code visually through diagram technology when it comes to building a program. Terminal user interfaces are their friend.

How to use

Verify node.js package manager: 
$ npm --version
Then install this scripting language: 
$ npm install --global inom
Read the manual for documentation: 
$ inom --help
Deploy this runtime backend to the internet: 
https://github.com:subvind/isisis
Download this desktop IDE frontend: 
https://github.com/subvind/autoide
